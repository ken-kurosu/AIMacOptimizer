## 1. エグゼクティブサマリ

### 1-1. プロダクト概要(確定ファクト)

- macOS 常駐メニューバーアプリ。メモリ/ディスク/ヘルス最適化 + 詳細診断 + ローカルAI相談。
- 直販(公証済みDMG、App Store外)。日本語/英語/中国語対応。
- Developer ID 署名・公証済み → Gatekeeper 警告なし。
- 料金: **Free**(メモリ最適化・タブ分析・診断・ローカルAI相談・ストレージスキャンが全部無制限) / **Pro**(ストレージのファイル削除/クリーンアップ + スケジュール自動最適化 + 優先サポート)。**月額¥480 / 買い切り¥4,980**。
- CTA: DMGダウンロード(直販) と Pro購入(Stripe Payment Link)。

### 1-2. バリュープロポジション案(3案)

| # | 一言VP | 狙い |
|---|---|---|
| **A(推奨)** | **「盛らない、Macの最適化。」** — 予測ではなく、実際に解放されたMBだけを表示する正直なMacオプティマイザ | 競合最大の弱点(過大な解放予測)への直球カウンター。1行で差別化が伝わる |
| B | 「数字で信頼できる、AI搭載のMacメンテナンス。」— 実測表示 × ランニングコスト0のオンデバイスAI | 実測 + AI の2軸を同時に立てる。ハイテク好きに刺さるが焦点がやや分散 |
| C | 「あなたのMacの主治医を、メニューバーに。」— 診断・最適化・相談まで、データを外に出さずに完結 | 情緒的で覚えやすい。ただし「正直・実測」の鋭さが弱まる |

**推奨はA**。ヒーローの主見出しに据え、サブコピーでB・Cの要素(AI、診断、プライバシー)を回収する。

### 1-3. ターゲット像

**ハイテク好きのビジネスマン**(30〜50代、M1〜M4 Mac利用、開発者・デザイナー・パワーユーザー含む)

- 効率・数字・先進性・信頼を重視。「解放できます: 12GB!」のような盛った数字には既に懐疑的。
- CleanMyMac の広告的トーンに疲れている層 / OSS(Stats等)は使えるが「削除系は怖い」層。
- サブスク疲れがあり、買い切り¥4,980 という選択肢に好意的。
- AI機能には興味があるが「データがクラウドに送られる」ことへの警戒が強い。
- 英語LPも読めるが、日本語で特商法まで揃った国産直販に安心感を持つ。

### 1-4. トーン&マナー

- **先進的で洗練、上質**。Apple公式サイト・Linear・Arcブラウザのような静的な自信。
- 誇張しない。「爆速」「劇的」「魔法のように」等の広告語彙は**禁止**。数字と仕組みで語る。
- 過剰なwidget・パーティクル・派手なネオンは避ける。動きは少なく、意味のある箇所にだけ。
- 一人称は使わず、断定調。「〜します」「〜しません」を明確に(特に「しません」= データを送らない/予測を盛らない、が本製品の個性)。

### 2-2. 市場の構造とAIMacOptimizerのポジション

市場は2極に割れている:

1. **「クリーナー系」**(CleanMyMac / MacBooster): 多機能・リッチLPだが、**数字を盛る**文化。解放予測の過大表示、検証不能な実績数字、強いサブスク誘導 → パワーユーザーの不信を買っている。
2. **「モニタ系」**(iStat Menus / Stats / Sensei): 正確・軽量だが、**見るだけで直せない**。最適化・削除・提案の機能がない。デザインも保守的。

**AIMacOptimizer は唯一この間を埋める**:

> **モニタ系の正確さ・軽さ・信頼 × クリーナー系の実行力 + どこにもないオンデバイスAI相談。しかも数字は実測のみ。**

### 2-3. 刺さるメッセージ(競合の弱点への対応マップ)

| 競合の弱点 | AIMacOptimizerの返し(LPで明言する) |
|---|---|
| CleanMyMac等の過大な解放予測 | 「予測は表示しません。**実際に解放されたMBだけ**を表示します」 |
| クリーナー系への「消してはいけないものを消しそう」という恐怖 | フォント/システム領域を保護・ゴミ箱経由(復元可)・項目別選択・リスク操作は個別承認 |
| AI系ツールのAPI課金・クラウド送信 | 「**APIキー不要・オフライン・追加費用0円**。データはMacから出ません」 |
| サブスク疲れ | Freeが本気で使える + 買い切り¥4,980 の明快な2択 |
| Sensei/iStatの価格不透明 | 価格をLPに**明記**(¥480/月・¥4,980買い切り)。比較表も載せる |
| モニタ系の「見るだけ」 | 監視 → 診断 → ワンクリック最適化 → AI相談まで1アプリで完結 |
| 野良アプリへのGatekeeper不安 | Developer ID 署名・公証済みを信頼バーで明示 |

## 3. 情報設計(ページ構成)

セクションは以下の順。番号 = DOM順。1ページ縦長LP、固定ヘッダー付き。

### S0. 固定ヘッダー

- **目的**: 常時CTA露出とセクションナビ。
- **構成**: 左にアプリアイコン+ロゴタイプ「AIMacOptimizer」。中央にアンカーナビ(機能 / 安全性 / 料金 / FAQ)。右に言語スイッチ(JA/EN/ZH)とプライマリCTA「無料ダウンロード」。
- **挙動**: スクロールで背景がガラス化(blur + 半透明)。モバイルはハンバーガー。

### S1. ヒーロー

- **目的**: 3秒で「正直な実測系オプティマイザ」と伝え、DMG DLへ誘導。
- **見出し(H1)**: 「盛らない、Macの最適化。」
- **サブコピー**: 「AIMacOptimizerは、解放"できるはず"の数字を見せません。実際に解放されたメモリとディスクの実測値だけを表示する、メニューバー常駐の正直なオプティマイザです。オンデバイスAI相談つき、追加費用0円。」
- **ビジュアル指示**: 画面右(モバイルは下)に**メニューバーのポップオーバーUIの実スクショ**(メモリゲージ+「実測 -842 MB 解放」のような結果表示状態)。スクショの上端にmacOSメニューバーを模したフレームを付け「常駐アプリ」であることを直感させる。背景は深いダークに繊細なグラデーションメッシュ(§4参照)。
- **CTA**: プライマリ「無料でダウンロード(.dmg)」+ 補助テキスト「Apple公証済み・警告なしで開けます / macOS 14+」。セカンダリはゴーストボタン「料金を見る」(#pricingへ)。
- **マイクロ実証**: CTA直下に小さく「Free版でメモリ最適化・診断・AI相談まで無制限」。

### S2. 信頼バー

- **目的**: DL前の不安(野良アプリ・怪しいクリーナー)を即時解消。
- **構成**: 横並び4アイテム(アイコン+短文)。
  1. 「Developer ID 署名・Apple公証済み」
  2. 「データはMacの外に出ません」
  3. 「削除はゴミ箱経由・復元可能」
  4. 「日本語 / English / 中文」
- **ビジュアル指示**: 淡いボーダーで区切ったストリップ。装飾なし・小さめタイポで"静かな信頼"。

### S3. 課題提起

- **目的**: 「クリーナー系の数字は信じられない/モニタ系は見るだけ」という共感を作る。
- **見出し(H2)**: 「"12GB解放できます" ── その数字、本当でしたか?」
- **本文コピー案**: 「多くのクリーナーアプリは、解放"できるはず"の容量を大きく見せます。実際にどれだけ軽くなったかは、誰も教えてくれません。一方でシステムモニタは正確ですが、眺めることしかできません。私たちは、その間を作りました。」
- **ビジュアル指示**: 左右2カラムの対比カード。「予測を盛るクリーナー(グレーアウトの抽象UI、数字に "?" を重ねる)」vs「AIMacOptimizer(実測 -842MB のような確定数字)」。競合名は出さない(抽象表現に留める)。

### S4. 主要機能① — 実測メモリ最適化(主役セクション)

- **目的**: 差別化の核「実測」を製品体験として見せる。
- **見出し(H2)**: 「表示するのは、実際に解放された数字だけ。」
- **本文**: 「ワンクリック最適化の結果は、最適化前後のメモリを実測した差分で表示します。予測が当てにならない操作では、そもそも数字を出しません。ディスクのクリーンアップも同じ ── 削除前後の実サイズ差だけをお見せします。」
- **ビジュアル指示**: セクション中央に**大型のメモリゲージ(円形orセミサークル)のアニメーションデモ**。「最適化」ボタン押下 → ゲージが下がる → 「-842 MB 解放(実測)」がカウントアップ表示、という一連の演出(§5参照)。実アプリのUIトーンに合わせる。
- **CTA**: なし(没入優先)。

### S5. 主要機能② — 機能グリッド

- **目的**: 網羅性を1画面で伝える(常駐モニタ+実行系+AI)。
- **見出し(H2)**: 「メニューバーに、Macの主治医を。」
- **構成**: 2×3 か 3×2 のカードグリッド。各カード = アイコン+機能名+2行説明。
  1. **プロセス可視化** — 何がメモリを食っているか、常駐で一目瞭然。
  2. **ストレージ分析** — DerivedData、Adobe・Cursorなどの大型キャッシュを自動検出。
  3. **ディスク自動ガード** — 空き容量が圧迫されたら、安全な提案または自動で対処。(自動実行はPro)
  4. **健康状態の推移** — メモリ/Swap/ディスク/CPUの推移をグラフで記録。
  5. **バッテリー健全度** — バッテリーの劣化状態をチェック。
  6. **詳細診断(9項目)** — Macの状態を9項目で総合診断。
- **ビジュアル指示**: カードはガラス調(subtle)。ホバーで微発光ボーダー。各カードに小さな実UIの切り抜きスクショを添えられればベスト(なければアイコンのみで成立する設計に)。

### S6. 安全設計

- **目的**: 削除系機能への恐怖を「設計思想」で解消する。Pro(ファイル削除)購入の心理障壁を下げる伏線。
- **見出し(H2)**: 「消す前に、理解と納得を。」
- **本文**: 「破壊的な操作は、内容を理解し、同意した上でだけ実行されます。」
- **構成**: 4項目のリスト(チェックアイコン+1行)。
  1. フォント・システム領域は保護され、対象になりません
  2. 削除はゴミ箱経由。いつでも復元できます
  3. 項目別に選択 ── 全部まとめて消させません
  4. リスクの高い操作は、1件ずつ個別承認
- **ビジュアル指示**: 実アプリの「削除確認ダイアログ」or「項目別チェックリスト」画面のスクショを右側に。安全 = 静けさ、で余白広めに。

### S7. プライバシー / コスト0のAI

- **目的**: 「AI搭載」の警戒(クラウド送信・従量課金)を反転させ、強みにする。
- **見出し(H2)**: 「AIに相談できる。データは、Macから出ない。」
- **本文**: 「ローカル解析とAppleのオンデバイスAIで動くため、APIキーは不要。オフラインでも動作し、あなたのMacの情報が外部に送信されることはありません。ランニングコストは0円 ── 何度相談しても、追加費用はかかりません。」
- **構成**: 3つの数字/記号を大きく見せるスタットバー: 「¥0 / 月(AI利用料)」「0件(外部送信データ)」「APIキー不要」。その下にAI相談UIのスクショ(チャット形式の実画面)。
- **ビジュアル指示**: このセクションだけアクセントカラー(§4のティール)を効かせ、"AIセクション"として視覚的に区別。ただしネオン過多にしない。

### S8. 軽さ(常駐の作法)

- **目的**: 「常駐アプリ=重い」の先入観を消す。ハイテク層に効く技術的誠実さ。
- **見出し(H3程度の小セクション)**: 「常駐しても、静かに。」
- **本文**: 「非表示中はポーリング頻度を自動で絞り、Macに負荷をかけません。最適化アプリ自身が重い ── そんな本末転倒はありません。」
- **ビジュアル指示**: S7の下に薄い帯として配置。左に小さなアクティビティモニタ風の低いグラフ線。

### S9. 料金(Free / Pro)

- **目的**: Freeの太さで信頼を稼ぎ、Proへ自然に誘導。価格透明性で競合(価格非表示のSensei/iStat等)と差別化。
- **見出し(H2)**: 「まず、無料で全部試せます。」
- **サブコピー**: 「Freeは体験版ではありません。メモリ最適化も、診断も、AI相談も、無制限。」
- **構成**: 2カラムのプライシングカード。
  - **Free ¥0**: メモリ最適化(実測表示) / タブ分析 / 詳細診断(9項目) / ローカルAI相談 / ストレージスキャン ── **すべて無制限**。CTA:「無料でダウンロード」
  - **Pro(推奨バッジ)**: Freeの全機能 + ストレージのファイル削除/クリーンアップ + スケジュール自動最適化 + 優先サポート。**月額¥480** または **買い切り¥4,980**(トグルor併記)。CTA:「Proを購入」(Stripe Payment Link)
  - 買い切り併記時の補助コピー: 「約10ヶ月でモトが取れる買い切り。サブスク疲れの方へ。」
- **ビジュアル指示**: Proカードをわずかに大きく・グラデーションボーダーで持ち上げ。価格数字は最大級のタイポで。「価格を隠しません」という姿勢自体が差別化なので、金額を目立たせる。

### S10. 比較表

## 6. 実装への技術メモ(Claude Design向け)

- **構成**: **単一HTML(`docs/index.html`)+バニラJS+CSS** を推奨。理由: 現行が単一HTML直販(おそらくGitHub Pages/静的ホスティング)で、ビルドパイプライン不要・LCP最速・保守が容易。Reactは不要(状態はi18n・トグル・アコーディオン程度)。
- **アニメライブラリ**: 上記仕様は**素のCSS + IntersectionObserver + rAFカウントアップ関数(30行程度)で全て実装可能**。ライブラリ不使用を第一候補とする。どうしてもタイムライン制御が煩雑ならGSAP(core+ScrollTrigger, CDN, ~70KB)を許容。Lottie/Framer Motionは不採用(単一HTMLに過剰)。
- **メモリゲージ**: SVGのcircle stroke-dasharrayで実装(canvasにしない)。数字はDOMテキスト。
- **レスポンシブ**: ブレークポイント 1120px / 768px / 480px。ヒーローは768px以下で縦積み(コピー→CTA→スクショ)。比較表はモバイルで横スクロール(`overflow-x:auto` + スクロールヒント)。タップターゲット44px以上。
- **パフォーマンス**:
  - 画像は全てWebP(+PNG fallback不要、macOSターゲットなのでWebP一択可)、`loading="lazy"`(ヒーローのみeager+`fetchpriority="high"`)。
  - スクショは1x/2xの`srcset`。合計ページウェイト目標 < 1.5MB。
  - フォント: Interは`font-display: swap`でwoff2をサブセット読込 or システムフォントのみでも成立させる。Noto Sans JP読込時は必ずサブセット。
  - LCP < 2.0s、CLS < 0.1 を目標。アニメはtransform/opacityのみ使用。
- **アクセシビリティ**: セマンティックHTML(header/main/section/footer、h1は1つ)。コントラスト比4.5:1以上(Text Secondary `#9AA3BC` on `#0B0E1A` は約7:1でOK)。アコーディオンは`<details>`or aria-expanded。`prefers-reduced-motion`対応必須(§5)。フォーカスリング可視。
- **多言語(ja/en/zh)**:
  - 推奨: **`?lang=` + `localStorage` + JSONの文言辞書によるクライアントi18n**を単一HTMLに内蔵(`<span data-i18n="hero.title">`方式)。デフォルトは`navigator.language`判定でja/en/zh自動選択、ヘッダーで手動切替。
  - `<html lang>`も動的に切替。SEO重視なら将来的に`index.html`/`en/index.html`/`zh/index.html`の静的3枚出し+`hreflang`が理想だが、初期リリースは1枚+クライアント切替で可(§9で確認)。
- **OGP/メタ**: `og:title`「AIMacOptimizer — 盛らない、Macの最適化。」/ `og:description`(サブコピー流用) / `og:image` 1200×630(§7) / `twitter:card` summary_large_image / `meta description`(120字) / favicon+`apple-touch-icon` / FAQPage JSON-LD / SoftwareApplication JSON-LD(offers: ¥0, ¥480/月, ¥4,980)。
- **計測(任意)**: プライバシー訴求と矛盾しないよう、入れるならCookieレスの軽量解析(Plausible/GoatCounter等)に限定し、フッターに明記。GAは非推奨。DLボタンに`?src=`程度のパラメータ付与でも可。**「データを送らない」アプリのLPがトラッカー満載、は絶対に避ける**。
- **CTAリンク**: DMGダウンロードURLとStripe Payment Link URLは定数としてHTML先頭付近にまとめ、差し替えやすくする(現時点でURL未確定、§9)。
